##GCT.R使用说明：

  • 代码实在R.objc上代码的修改，思路类似于R.Swift一样

###CGT.R使用步骤：添加之前配置一下自己的xcode, 让xcode遇到错误的时候继续编译，勾选: Perferences->General->Continue building after errors

  1. pod 'GCT.R
  2. TARGETS -> Build Phases -> New Run Script Phase 添加下面的命令"${PODS_ROOT}/GCT.R/GCT.R/GCTR" -p "$SRCROOT"
  3. build或者run项目，你会发现多了个R.h和R.m两个文件，拖进项目中，不要勾选Copy items if needed

GCTTheme和GCT.R配合使用

  • 两个东西的作用是相对独立，也有一定的覆盖：
  1. GCTheme赋值切换皮肤（这里包裹图片、文字，颜色，这里都是本地资源），后期会加入服务端资源，以及其他类型的资源文件。
  2. 配置的时机，需要切换，尽量不要放到AppDelegate中，影响启动时间。已经通过GCT.R延时处理了
  3. ！！都要通过字符串去查找，编码不是很友好！（后期会通过R.Obj去修正这个问题。） 已经通过GCT.R处理了
  4. GCT.R代码的功能，就是在编译的时候，生成一系列的类、属性，从而避免通过字符串的方式来获取图片、文字等资源文件[后期这个也可以做扩展、storyboard/xib/本地strings/多语言/基本校验代码功能]
